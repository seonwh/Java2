## 03/18
**[기계어]** 
* 0, 1의 이진수로 구성된 언어. 
* 컴퓨터의 CPU는 기계어만 이해하고 처리가 가능함. 

**[어셈블리어]** 
* 기계어 명령을 ADD, SUB, MOVE 등과 같은 표현하기  쉬운 상징적인 단어인 **니모닉 기호 (mnemonic symbol)** 로 일대일 대응시킨 언어.

**[고급 언어]** 
* 사람이 사용하는 언어로 이해하기 쉽고, 쉽게 표현할 수 있도록 고안된 언어.
* Pascal, Basic, C/C++, Java, C# 
* *절차 지향 언어와 *객체 지향 언어로 나뉜다.

**[절차 지향 언어]**
* 프로그램을 절차, 순서에 따라서 실행하는 방식.
* 데이터(입력)와 함수를 분리하여 작성함.
* 코드의 유연성이 부족, 재사용이 어려움.
* 전역 변수를 많이 사용, 때문에 코드의 가독성과 유지보수가 어려움.
* C, Pascal, Fortran 등이 있음.

**[객체 지향 언어]**
* 현실의 객체를 모델링하여 프로그램을 작성하는 방식.
* 객체는 데이터와 데이터를 처리하는 메소드(함수)를 모두 포함함.
* 상속, 캡슐화, 다형성 등의 개념을 활용, 유연하고 재사용 가능한 코드를 작성가능.
* Java, C++, Python 등이 있음.

**[함수 지향 언어]**
* 함수를 일급 객체를 취급, 상태 변경을 피하고 불변성을 지향하는 프로그래밍 패러다임.
* 함수의 조합으로 복잡한 작업을 수행, 상태 변경 대신 데이터를 변환하는 방식으로 프로그램을 작성함.
* 재귀, 고차 함수, 순수 함수 등의 개념을 중요시하고, 병렬 처리와 높은 수준의 추상화를 지원.
* 객체 지향 언어의 특성을 그대로 가져가되, 함수 지향적인 기능을 추가한 것.
* Python, Kotlin, Haskell, Lisp, Scala 등이 있음.
