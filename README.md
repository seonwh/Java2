**[참조 자료형]**
* 포인터는 임의의 메모리 주소를 저장 하는 반면, 참조 자료형은 주소를 저장 못함.
* 직접 주소를 갖고 있지는 않지만 JVM이 해당 주소로 안내함.
* 객체를 참조하는 변수 유형, *Heap 영역에 저장된 객체의 메모리 주소를 가리킴.
* 새 키워드로 객체를 생성하는 것이 참조 자료형을 선언.
* Array, Interface, Enum도 객체이기 때문에 참조 자료형.
* 객체를 참조하지 않을 때 null을 가질 수 있다.
* 같은 객체를 여러 변수가 참조할 수 있고, == 연산자로 객체의 주소를 비교 할 수도있다.

**[메모리 구조]**
* 힙(heap -FIF0) 영역은 프로그래머가 직접 공간을 할당, 해제하는 메모리 공간.
* 스택(stack - LIF0) 영역은 프로그램이 자동하는 사용하는 임시 메모리 영역.
* 힙이 스택을 침범하는 경우를 힙 오버 플로우, 스택이 힙을 침범하는 경우를 스택 오버 플로우라고 한다.

**[변수 선언]**
* 변수 : 프로그램 실행 중에 값을 임시로 저장하기 위한 메모리의 공간 프로그램 수행중 변경 될 수 있음.

**[상수 선언]**
* final 키워드 사용.
* 선언할 때 초기값 지정.
* 실행 중 값의 변경 불가.

**[var]**
* Java 10부터 도입됨.
* 타입을 생략하고 변수 선언을 할 수 있음.
* 컴파일러가 추론하여 변수 타입을 결정.
* 변수 선언할 때 초기값을 주지 않으면 컴파일 오류가 발생.

  * 지역 변수: 메소드 내부에 선언되는 변수.
  * 클래스 필드: 클래스 내부에 선언되는 변수, 객체가 생성될 때 함께 만들어지는 변수.


* 명시적 자료형을 사용하는 것이 좋음.
* 가독성이 유지될 수 있는 한에 적절히 활용할 것.
* 상수를 적극적으로 활용해, 코드의 안정성을 높일 것.

### System.out.print
**System.out.print()**
* 기본 출력문, 줄 바꿈을 하지 않고 한줄로 출력됨.
* 줄 바꿈을 하려면 개행 문자 \n을 추가로 넣어야 함.

**System.out.println()**
* 출력 후 자동으로 줄 바꿈을 실행함.
* 개행 문자 없이 자동으로 줄 바꿈이 됨.

**System.out.printf()**
* 형식을 지정하여 문자열을 출력할 때 사용함.
* 값을 특정한 형식(소수점 자리 수, 정렬 등)으로 출력할 때 유용함.

### [타입 변환]
* 특정 데이터 타입의 값을 다른 데이터 타입의 값으로 변환
* **자동 타입 변환**
  * 컴파일러에 의해 원래의 타입보다 큰 타입으로 자동 변환
  * 치환문이나 수식 내에서 타입이 일치하지 않을 때
* 강제 타입 변환

  * () 안에 개발자가 명시적으로 타입 변환 지정
  * 강제 변환은 값 손실 우려
* 큰 데이터 타입에서 작은 데이터 타입으로는 변환 되지 않음.

### WORA
**WORA (Write Once Run Anywhere)**
* 학번 작성된 코드는 OS, H/W에 상관없이 모든 플랫폼에서 바로 실행된다.
* C/C++ 등 기존 언어가 가진 플랫폼 종속성 극복
* 네트워크에 연결된 어느 클라이언트에서나 실행. 웹 브라우저, 분산 환경 지원 가능.

이를 가능케 하는 자바의 특징
* 바이트 코드 (byte code)
  * 자바 소스를 컴파일한 목적 코드
  * CPU에 종속적이지 않은 중립적 코드
  * *JVM에 의해 해석되고 실행

1. [x] JVM(Java Virtual Machine)
    * 자바 바이트 코드를 실행하는 자바 JVM(소프트웨어)

[ 바이트 코드 (.class) ]
* 자바 JVM 에서 실행가능한 바이너리 코드
* 바이트 코드는 컴퓨터 CPU에 의해 직접 실행되지 않음
* 자바 JVM이 작동 중인 플랫폼에서 실행
* 자바 JVM이 인터프리터 방식으로 바이트 코드 해석

[ JVM (Java Virtual Machine) ]
* 각기 다른 플랫폼에 맞는 JVM을 제공, 이 자체는 플랫폼에 **종속적**.
* 각기 다른 플랫폼에서도 동일한 자바 실행 환경을 제공.
* 자바 개발사인 오라클 외, IBM, MS 등 다양한 회사에서 제작 공급.
* 자바의 실행은 JVM에서 클래스 파일(.class) 을 바이트 코드 실행 하는 것.

### JDK와 JRE
JDK의 bin 디렉터리에 포함된 주요 개발 도구
* javac : 자바 소스를 바이트 코드로 변환하는 **컴파일러**
* java    : 자바 응용프로그램 실행기. **JVM을 작동시켜 실행**
* javadoc : 자바 소스로부터 HTML 형식의 API 도큐먼트 생성
* jar     : 자바 클래스들(패키지 포함)을 압축한 아카이브 파일(.jar) 생성 관리
* jmod    : 자바의 모듈 파일(.jmod)을 만들거나 모듈 파일의 내용 출력
* jlink   : 응용프로그램에 맞춘 맞춤형(custom) JRE 제공
* jdb     : 자바 응용프로그램의 실행 중 오류를 찾는 데 사용하는 디버거
* javap   : 클래스 파일의 바이트 코드를 소스와 함께 보여주는 디어셈블러

Java SE (Standara Edition)
* 데스크톱, 서버, 임베디드 시스템을 위한 **표준 플랫폼**
* `java.lang.*, java.util.*, java.awt.*, javax.rmi.*, javax.net.*` 등 대부분의 패키지가 포함되어 있는 대중적 플랫폼

Java EE (Enterprise Edition)
* 자바를 이용한 **서버측 개발을 위한 플랫폼**
* `JSP, Servlet, JDMC, JNDI, JTA, EJB` 등 웹 프로그래밍에 필요한 기능을 포함하고 있는 플랫폼이며, Java SE 플랫폼을 기반으로 그 위에 탑재

Java ME(Micro Edition)
* **임베디드**를 위한 자바 플랫폼
* `휴대전화, PDA, 셋톱박스` 등에서 Java프로그래밍 언어를 지원하기 위해 만들어진 플랫폼이며, Java SE 플랫폼을 기반으로 그 위에 탑재
